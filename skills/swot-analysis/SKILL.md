# SWOT Analysis

Produce a SWOT that earns its meeting slot: evidence-backed entries, cross-quadrant strategy moves, and a shortlist of decisions. The four boxes are the input; the TOWS moves are the deliverable.

## Workflow

1. **Set the frame.** A SWOT of "the company" produces mush. Fix the subject and scope with the user: which business/product, which market, and what decision this analysis feeds (enter a market? raise prices? annual plan?). Internal factors (S, W) are things the org controls; external (O, T) are things it doesn't — police this boundary, it's the most common SWOT error.
2. **Populate with evidence, ruthlessly.**
   - Each entry: one line + the evidence for it (metric, customer quote, market fact). No evidence → goes in an "unverified beliefs" list instead, not the matrix.
   - Strengths must be *relative to competitors* ("48-hour implementation vs. industry 6 weeks"), not table stakes ("good team").
   - Cap each quadrant at 5 items, ranked. Ten-item quadrants mean no prioritization happened.
3. **Do the TOWS crossing — this is the actual analysis.** Generate moves from pairings:
   - S×O: which strength captures which opportunity? (offensive plays)
   - S×T: which strength neutralizes which threat? (defensive plays)
   - W×O: which weakness must be fixed to unlock an opportunity? (investment plays)
   - W×T: where do weakness and threat combine? (survival risks — flag these loudest)
   Write 2–3 concrete moves per pairing that name the specific S/W/O/T items involved.
4. **Converge on recommendations.** From the moves, pick the top 3 with: rationale, first step, owner placeholder, and the tradeoff being accepted. Explicitly list what this implies saying *no* to — strategy without exclusions is a wish list.

## Output format

Context line (subject, scope, decision) → SWOT matrix (evidence in-line) → TOWS moves grouped by type → Top-3 recommendations → Unverified beliefs to test. Keep the whole thing under two pages.

## Quality bar

- Zero generic entries: if a competitor could paste the same line into their SWOT, cut or sharpen it.
- Threats include probability/timing when statable ("likely within 12 months"), not just existence.
- The analysis must surface at least one uncomfortable weakness; all-flattering SWOTs are decoration.
